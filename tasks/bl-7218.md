Today --status ready|blocked|claimed filters the live ladder, while --closed/--all are a SEPARATE axis (Reach::Dead/All in src/reads/flags.rs) picking which commits to read. Two flag idioms for one question ('which status?'). The reach split leaks an implementation detail (live tree scan vs history walk) into the interface.

Fold them into one axis. --status accepts every rung: ready|blocked|claimed|closed (no 'dropped' once the sibling lands). Reach is INFERRED from the requested rung — asking for 'closed' triggers the dead-balls history walk; the live rungs stay a tree scan. --all stays = no status filter (live+dead). --closed retires, subsumed by --status closed.

Add -s as the short alias for --status. Target ergonomic: `bl list -s closed`.

Scope: src/reads/flags.rs (parse_status accepts 'closed' + infers reach; -s alias; drop set_reach/--closed or keep --all only; fix the 'want ready|blocked|claimed' error text), list.rs, SKILL.md command table. Needs the dropped→closed collapse first so 'closed' is the single terminal rung.
