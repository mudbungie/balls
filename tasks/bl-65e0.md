drop is a composite, not a primitive: drop = unclaim (clear occupancy, release the worktree) ∘ close (delete the file, archive). Retire carries both verbs but they differ only in which guard runs — and that difference is a §10 enforcement hole: a `--blocks close` gate is unenforceable while drop exists, because dropping deletes the file exactly like closing it, gate unchecked. Killing drop closes the hole and shrinks the deliverable verb set to five.

Replacement flows:
- abandon a held task: `bl unclaim X` then `bl close X` (uncommitted worktree WIP dies at unclaim, same as drop; the empty deliverable makes close's delivery a no-op).
- retire an unheld task: `bl close X` (already the documented path).

Deliberate semantic delta: committed WIP on work/<id> survives unclaim (branch deletion is deferred to prime, §14), so a later close DELIVERS it where drop discarded it. That is close honoring its contract — work committed to the branch is work; discarding it now takes an explicit `git branch -D work/<id>`.
