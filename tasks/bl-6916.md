architecture.md §13:1083-1088: 'bl sync --branch landing is a no-op — for free, not by special-case… The general rule fetch a branch's upstream, if any yields nothing on an upstream-less branch.' The code is the forbidden shape: src/checkout.rs:126-128 — if opts.branch.as_deref() == Some("landing") { return Ok(()); } — keyed on the literal TOKEN, not the actual branch. So bl sync balls/config takes the general path, where the tracker fetches origin:balls/config and tries merge --ff-only FETCH_HEAD into the store's checked-out branch (src/tracker/remote_ops.rs:25-27) — fails non-ff. Outcome is harmless (the landing ref never moves, §4 consent holds) but by accident, not structure. Also: spec spells the arg --branch, code takes a positional.

Fix: delete the special case; make the general rule ('fetch the branch's configured upstream, if any') actually yield the no-op for the landing. Reconcile the --branch/positional spelling whichever direction is true (§15 entry if the spec side moves).

**Why:** 'a special case is usually a missing reframe' — the spec already did the reframe and the code didn't follow. The literal-token key is also a latent bug: the guard guards the wrong name.
