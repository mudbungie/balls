PROBLEM: 'bl claim' materializes the work/<id> worktree but does NOT print its path, and 'bl show --json' deliberately omits the stored 'delivery-worktree' key (the SKILL documents both). So the very next thing an agent must do after a claim — cd into the worktree to edit — requires a side query: 'git worktree list | awk "/work\/<id>/{print $1}"'. The one fact you need back from claim is the one fact it withholds.

FIX OPTIONS (pick one): (a) 'bl claim' prints the worktree path to stdout (it is the only natural stdout product of the verb, mirroring how 'create' prints the id); or (b) 'bl show' surfaces 'delivery-worktree' in the human projection (keep --json as the pure bedrock). Either closes the claim→edit loop without a git side-channel.

Observed during bl-1b49 E2E demo.
