Tripped live during bl-3914 verification: the §4 ladder is debug/info/error — there is no warn rung — and Level::parse (src/log.rs) reads any unrecognised value leniently as info. So `bl --log-level warn` (a value every other CLI accepts) is a SILENT no-op: the user asked for quieter output and got the default, with no signal anything was wrong.

FIX DIRECTION (pick one, subtraction-first): (a) strict parse — an unrecognised level is a usage error naming the ladder (fail loud beats lenient-wrong; no new mechanism); or (b) accept warn as an alias for error (one match arm; matches user muscle memory but grows the surface). Default recommendation: (a). SKILL.md already documents the no-warn gotcha (bl-3914) — whichever lands, update that line.
