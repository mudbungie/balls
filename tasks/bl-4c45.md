Two SS6 install claims the binary does not implement (the third found alongside - publishing via --to <center> - is the already-tracked open design leg bl-b8d6, scoped OUT of this ball):

- "Defaults: --to the landing, --from the configured upstream": bare `bl install` -> `--from <ref> is required` (source comments this as deliberate: core resolves no implicit upstream).
- Binary resolution "(PATH or explicit --bin <name>=<path>)": `--bin` -> `unexpected flag`; resolution is beside-bl only, never PATH.

Everything else in SS6 install holds (mirror/union shapes, siblings untouched, bin/ never travels, N added / M deleted, protocol validation incl. version refusal). Resolve each: wire it or strike it via a SS15 revision entry. Note the protocol-validation refusal currently fires AFTER the copy seals (the install commit lands, then exit 1) - decide whether that ordering is intended while here.
