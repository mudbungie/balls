architecture.md:380-381: 'bl plugin <name> <op> <phase> is the canonical dispatch and balls dogfoods it.' Neither half is real: no such verb exists (Verb::ALL, src/verb.rs:49-62; bl plugin → 'unknown command', exit 2), and core spawns plugins directly as <bin> <op> <phase> (src/plugin.rs:202-214) rather than through any verb. The load-bearing property — subprocess uniformity — DOES hold; what's missing is the user-invocable entry point and the dogfooding claim.

Either implement the verb (a thin wrapper over the existing spawn path — useful for hand-testing a plugin wiring) or strike the sentence with a §15 revision-log entry. Lean subtraction: if no concrete consumer exists, strike; the spec shouldn't carry an aspirational verb.

**Why:** the freeze discipline ('substantive design changes need a §15 entry — never a silent edit') cuts both ways: a frozen spec asserting a verb that exits 2 is a broken promise to anyone building against §6. Resolving it either direction is cheap; shipping 0.5.0 with it unresolved is not.
