Residual of the bl-2d6d security review MED-2 (docs/security-review-subprocess-git-paths.md). bl-23ac already host-bound the stored token (a changed api_base fails closed, re-auth required) and closed the token.json chmod TOCTOU (0600 create, not write-then-chmod). The remaining sliver: RepoConfig::validate() (crates/balls-github-shared/src/config_base.rs) checks only the repo owner/name shape; api_base accepts any scheme, so a user who runs auth-setup against an http:// base binds the token to it and then ships Bearer <token> cleartext on every call.

FIX: validate() rejects a non-https:// api_base, with a loopback carve-out (http://127.0.0.1 / http://localhost) so the mock-server tests and local dev keep working. Also warn on stderr when api_base differs from the github.com default, so a redirected base is a visible decision, not a silent config diff (the review's defense-in-depth ask).
