The docs/SPEC-clone-layout.md / SPEC-forge-gated-delivery.md / SPEC-lifecycle-sync-participants.md / SPEC-orphan-branch-state.md / SPEC-tracker-state.md family predates the frozen docs/architecture.md and has drifted: e.g. SPEC-lifecycle-sync-participants.md:308 references src/plugin/mod.rs ('run_plugin_push and run_plugin_sync in src/plugin/mod.rs become thin wrappers…') — that path doesn't exist (it's src/plugin.rs), and it names commands/lifecycle.rs and ball bl-fb4d. architecture.md §0-§17 is the single authoritative spec.

Fix: delete the family, or stamp each with a one-line superseded-by-architecture.md header if any still carries un-migrated design rationale worth keeping. Post-release is fine — these don't ship in the crate (docs/ isn't excluded, actually: verify whether docs/ ships in the package and weigh that in).

**Why:** two specs is no spec — every reader who lands on a SPEC-*.md first builds on the pre-freeze model. Subtraction is the fix; the freeze already decided which document is true.
