PROBLEM: 'bl create x -p1' fails with "bl create: unexpected flag '-p1'"; you must write '-p 1' (space-separated). Most CLIs (and getopt convention) accept the glued short-option form '-p1'. This cost a failed run in the bl-1b49 race-probe script before I spotted it. Low severity, pure papercut — but it violates the principle of least surprise for anyone with muscle memory from git/ls/etc.

FIX: accept the glued form for value-taking short flags (-pN, and any others). Verify the arg parser handles '-p1' identically to '-p 1'.

Observed during bl-1b49 E2E demo.
