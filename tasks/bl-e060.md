Follow-up to bl-a476. After origin discovery moved to the tracker, core's `checkout::resolve_remote` collapsed to `cli.or_else(|| xdg_remote(user_config))` — a trivial explicit-config read whose name still claims core 'resolves the remote', but resolution (incl. origin discovery) is the tracker's now (§0/§12). Delete the function: bind() inlines `cli_remote.or_else(|| xdg_remote)`; mutate (cli always None) reads `xdg_remote` directly, making 'core only reads explicit config' obvious. Move the xdg_remote Some/None coverage to a direct config test. Fix the dangling `[resolve_remote]` doc links.
