rm -rf a claimed task worktree dir, then `bl prime`: exit 1 with `git worktree add: fatal: <path> is a missing but already registered worktree; use add -f ... or prune ...` -> `plugin bl-delivery aborted the op`. The worktree is not recreated and the whole prime fails; recovery needs a manual `git worktree prune` in the project repo.

SS11 promises prime.post "re-materialize each still-claimed worktree", create-if-absent, "every hook is idempotent by construction". A deleted dir with a stale git registration is the ordinary form of "absent" (crashes, tmp cleaners, humans). Fix: bl-delivery materialize should prune the stale registration (or add -f) before worktree add.
