Build the adversary close.pre review gate: a single binary that, at task close (wired ahead of bl-delivery via 'bl conf prepend close.pre adversary'), runs a single-pass non-interactive claude over the task's diff (git diff main...HEAD) and its ball (the §7 payload's bl-id), and exits non-zero to abort delivery on a failed review. Core reads only the exit code; the review write-up rides stderr.

Design is settled and lives in docs/design.md (the living artifact — edit it like code as the design is attacked). The scaffold is committed and inert (the binary abstains/exits 0 on every op-phase), so the capability is added by replacing the dispatch arm, not flipping a default.

Open questions to settle before/while implementing (see design §5):
- verdict wire: how claude's answer becomes an exit code (narrowest parse that can't false-pass)
- model / effort / review rubric / context budget (single-pass = one shot)
- failure-mode policy: fail-open vs fail-closed when claude is unreachable (a knob in the plugin's own config territory, never a core field)

Deliverable: a working, 100%-covered close.pre plugin under the repo's pre-commit gate.
