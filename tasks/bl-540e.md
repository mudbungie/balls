architecture.md promises the flag in four places — §12:928 'Implicit founding is fine: creating a balls branch on a repo you can push to is harmless and once-per-clone — --stealth opts out (and locks the store local)', plus :924-925, :978-979, §15:1604 — but parse_prime (src/checkout.rs:215-228) accepts only --as/--remote/--center/--install. Stealth today is only ever INFERRED (src/tracker/mod.rs:120-132: no explicit remote AND no discoverable origin; src/tracker/prime.rs:33-39 then writes the self-lock).

Implement: bl prime --stealth takes the same path as the inferred case explicitly — write the self-lock, never found/push a remote branch. Decision confirmed by maintainer 2026-06-09: implement, do not strike.

**Why:** in any clone with a pushable origin, bl prime founds and pushes balls/tasks with no way to decline short of uninstalling the tracker or hand-editing the hook schedule. The spec's own 'implicit founding is fine' argument is explicitly CONDITIONED on the opt-out existing — without the flag, the §12 consent story is unsound, not merely incomplete.
