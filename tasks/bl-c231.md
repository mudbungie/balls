WHY: promote bl-430e from incident to principle. Every plugin effect is either BINDING — one
atomic, detectable commit point (the [bl-id] squash, the store seal, the push); retry detects
it and skips — or NON-BINDING — derived and overwritten on retry (the §14 DERIVE pattern;
merge --abort leaves nothing; capture/fold artifacts recompute). Half-merged states become
immaterial BY CONSTRUCTION, never a state to fear. Tiers 1-2 then need no rollback protocol
at all: converge-on-retry covers them. Rollback remains ONLY for effects whose binding
artifact lives in an EXTERNAL system keyed to an op that never sealed (the jira orphan
ticket: a retry mints a new id, so nothing ever converges onto the orphan — someone must
delete it). §14 inverts: converge-on-retry is the rule; rollback is the short appendix for
external effects. Code barely changes; conceptual surface shrinks.

VERIFIED knot-free (no git-surgery states), with two findings folded in:

(a) The reframe REMOVES today's scariest reset. §11's un-squash rollback (git reset --hard
HEAD~1 on the integration branch) races concurrent main movement in a shared hub — a sibling
receive between squash and rollback and the reset eats THEIR commit (§14's check-the-[bl-id]-
tag-first mitigates; dropping un-squash eliminates). A standing squash without a sealed close
is exactly the bl-430e state; retry completes it. Note the squash is always GATED code (the
gate runs before the squash), so a standing delivery is never unreviewed work.

(b) ONE real knot found — needs a guard, included in scope: the bl-430e skip reads
"[bl-id]-since-fork => already delivered", so commits added to work/<id> AFTER a delivered-
but-unsealed close are silently STRANDED by the retrying close's skip (then pruned with the
branch). Guard: [bl-id]-since-fork AND unmerged commits on work/<id> => ABORT — "already
delivered; work/<id> carries new commits — file a new task or deliver manually". Without the
guard, the reframe inherits a silent-loss edge; with it, the loud-failure discipline holds.

Interplay: the squash-path work shares ground with bl-a04a (fold rigor); land bl-a04a first.

Deliverable: spec §14 (+ §11 rollback rows) + §15 entry; delivery plugin + engine + tests.
