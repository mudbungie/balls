Design task; the deliverable is the living doc `docs/completion-gate.md` (edit it like code as the design is attacked). This ball records the work; the file is the artifact.

Generalizes bl-d014 from a fixed *adversarial review* gate into a **rubric-agnostic `close.pre` LLM gate**: the repo owner supplies the rubric, so "completion vs quality" is config, not a core opinion. Came out of a design dialogue comparing balls to Codex `/goal` mode — the gate plus a harness loop reconstruct goal-mode outcomes, but with a deterministic stop condition (a balls query, not a self-audit) and a separate-process verifier (fresh context by construction, which solves grade-the-grader).

See the doc for: the gate contract (pure verdict, disposition lives in the consumer); the two holes (Hole 1 deterministic-vs-judgmental — facts stay on the pre-commit hook, the LLM judges only what judgment can settle and cites artifacts; Hole 2 grade criterion->intent, not just diff->criterion); the harness loop + runaway guard ("keep going" is the harness, not balls and not the plugin); handoff = unclaim + body/journal note (commit WIP first), new task only when the remainder is separable; and the structure-enforcement plugin as the gate's optional, severable input contract. Watch items: criterion->intent grading leaks quality back into a completion-only gate; splitting-to-pass is the gaming pattern in a follow-up costume.
